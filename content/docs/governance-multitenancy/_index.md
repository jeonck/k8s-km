조직 단위의 정책·격리·규정 준수를 다루는 영역이다. 개별 리소스의 보안 설정(계층 2의 보안)과는 결이 다르다 — 여기서는 "여러 팀/조직이 하나의 클러스터를 안전하게 나눠 쓰는 규칙"과 "조직 표준을 어떻게 코드로 강제하는가"를 다룬다.

| 어젠더 | 핵심 질문 |
| --- | --- |
| 네임스페이스 기반 테넌트 격리와 한계 | 네임스페이스만으로 얼마나 격리되는가 |
| 정책 거버넌스 (조직 표준의 코드화) | 조직 규칙을 어떻게 자동으로 강제하는가 |
| 감사 로그 (audit) | 누가 무엇을 했는지 어떻게 남기는가 |
| 규정 준수 프레임워크 매핑 | 외부 컴플라이언스 요구를 클러스터 설정에 어떻게 연결하는가 |
